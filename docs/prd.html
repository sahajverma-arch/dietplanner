1. Overview

Dietitians run 45–50 minute first-counselling calls where they collect a client's food preferences, lifestyle, and medical history — then spend significant additional time manually composing a personalised 1-week diet chart, formatting it, and repeating a lighter version of this every week per client. The LEANR Diet Platform removes the composition and formatting work: the dietitian fills a structured form live during the call, and the platform generates a restriction-aware, macro-annotated 7-day plan as a branded, client-ready PDF in about a minute. Weekly follow-ups take a short check-in form and produce the next week's adjusted plan.

2. Goals & Non-Goals

Goals

  • Cut per-client plan preparation time from hours to minutes while keeping the dietitian in control of all inputs.
  • Guarantee plans respect hard constraints: diet type (veg / non-veg / vegan / eggetarian), allergies, intolerances, dislikes, and medical conditions.
  • Produce a premium, client-shareable PDF with per-meal calories and macros, daily totals, and a weekly summary.
  • Zero data leakage between dietitians — every record is isolated at the database level.
  • Never lose intake data: the counselling form autosaves continuously during the live call.

Non-Goals (v1)

  • No client-facing portal or login — the dietitian is the only user; plans reach clients as PDFs.
  • No payment/subscription management, scheduling, or CRM features.
  • No medical diagnosis — the AI adapts plans to stated conditions but the dietitian reviews every plan.

3. Users

Primary persona — the practising dietitian. Handles 20–100 active clients, runs multiple counselling calls per day, needs to capture details fast mid-conversation, and sends every client a weekly chart. Values speed, correctness of restrictions, and a professional-looking deliverable carrying the LEANR brand.

4. User Journeys

4.1 First counselling (during the call)

  1. Dietitian signs in and clicks + New Counselling.
  2. Fills the 5-section form (Basics · Food Preferences · Lifestyle · Medical · Notes) while talking; every change autosaves with a visible "Saved" indicator. Closing the tab and returning restores the draft.
  3. Clicks Generate Week 1 Diet Plan. The AI composes the plan (~1 min), the PDF is stored, and the person becomes a saved client. The dietitian lands on the client's page with the plan and its downloadable PDF.

4.2 Weekly follow-up

  1. Dietitian opens the client card, records weight, adherence, energy, hunger, complaints, and notes.
  2. Clicks Generate Week N Plan — the AI sees the full intake, the check-in, and last week's menu (for variety) and produces the adjusted week with a fresh PDF added to the plan history.

5. Functional Requirements

IDRequirementNotes
FR-1Email/password authentication; a dietitian profile is created automatically on signup.Supabase Auth + DB trigger
FR-2Dashboard lists the dietitian's own clients as cards (goal, diet type, weight, current week, last plan date) with a "+ New Counselling" action.RLS-scoped query
FR-3Multi-section first-counselling form: basics, food preferences (incl. diet type, likes/dislikes/allergies/intolerances), lifestyle, medical (condition checklist, medications, labs), free-text notes.5 tabs, single state
FR-4The counselling form autosaves continuously (debounced ≤2s) server-side and restores on reload; the draft is cleared only after successful plan generation.form_drafts table
FR-5On submit, a strict-JSON 7-day plan is generated server-side honouring all restrictions, with per-meal calories, protein, carbs and fat.NVIDIA NIM + Zod validation
FR-6Every plan is rendered as a branded PDF (macro-report format: per-day meal tables, daily totals, weekly summary) stored in private cloud storage and downloadable via expiring signed URLs.@react-pdf/renderer
FR-7Submitting a first counselling converts the person into a saved client; intake is retained in full for future weeks.clients.intake JSONB
FR-8Weekly follow-up form (weight, adherence, energy, hunger, complaints, notes) generates the next week's plan; client weight is kept current.followups table
FR-9Client page shows the latest plan inline (meals, macros, guidelines), the full plan history with PDF downloads, and the follow-up form.
FR-10If generation fails after the client is saved, the intake is preserved and a one-click retry is offered on the client page.regenerate endpoint

6. Non-Functional Requirements

IDRequirement
NFR-1Isolation: Postgres Row Level Security on every table (dietitian_id = auth.uid()) and folder-scoped storage policies; one dietitian can never read another's data, even with the public API key.
NFR-2Secrets: the NVIDIA API key is server-side only and never shipped to the browser.
NFR-3Performance: plan generation typically completes in ≤90 seconds; hard cap 300 seconds with explicit failure and retry path.
NFR-4Resilience: per-call AI timeouts, automatic fallback model when the primary is congested, constrained JSON decoding, schema validation with one corrective retry, and split (two-call) generation to respect provider output limits.
NFR-5Data safety: no intake data loss on network failure, generation failure, or tab close (autosaved drafts + preserved clients).
NFR-6Deployability: single-command Vercel deploy; entire database provisioned by one SQL migration.

7. Success Metrics

  • Median time from "form complete" to "PDF in hand" ≤ 2 minutes (target ~1 minute).
  • ≥ 95% of generation attempts produce a valid plan (including automatic retries/fallback).
  • Zero cross-tenant data access incidents.
  • Zero reported intake-data losses during live calls.

8. Future Scope (v2 candidates)

  • WhatsApp/email delivery of the weekly PDF directly to the client.
  • Meal-level swap suggestions ("replace this meal") without regenerating the full week.
  • Client progress analytics (weight trend, adherence history) on the client page.
  • Plan templates and a dietitian-editable food database for tighter macro accuracy.
  • Multi-dietitian clinics with team/admin roles.
Review policy: every AI-generated plan is presented to the dietitian before it reaches a client; the PDF footer states the plan was generated with AI assistance and reviewed by the dietitian, and is not a substitute for medical advice.
